SMS Configuration Section shows:

═══════════════════════════════════════════════════════════════════
                    HOW TO CONFIGURE SMS:
═══════════════════════════════════════════════════════════════════

STEP 1: Open/Create .env file in project root

STEP 2: Add these lines:
   SMS_PROVIDER=africas_talking
   AFRICAS_TALKING_API_KEY=your_key_here
   AFRICAS_TALKING_USERNAME=sandbox

STEP 3: In Admin, check ✓ "SMS Enabled"

STEP 4: Test with: python test_sms.py

That's it! No more database fields to fill in.

═══════════════════════════════════════════════════════════════════
                    SECURITY IMPROVEMENT:
═══════════════════════════════════════════════════════════════════

Old Way (Database):
   ❌ API keys stored in database
   ❌ Visible in admin panel
   ❌ Included in database backups
   ❌ Hard to rotate credentials

New Way (.env file):
   ✅ API keys in environment variables
   ✅ Not in version control
   ✅ Not in database backups
   ✅ Easy to rotate credentials
   ✅ Follows security best practices
